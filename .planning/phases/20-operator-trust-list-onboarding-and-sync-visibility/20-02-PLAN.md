# Phase 20 Plan 20-02: Guided Trust-List Onboarding UI, Timeline, Docs, and Final Validation

**Created:** 2026-04-23
**Status:** Ready for execution

## Objective

Turn the trust-list admin page into an operator-friendly guided workflow with optional source testing, sync timeline/status visibility, prescriptive recovery guidance, localized copy, and operator documentation.

## Scope

### In Scope

- Add a guided wizard on existing `/admin/trust-lists`.
- Show optional pre-save preview/test result and warn when saving without test.
- Show per-source sync timeline/status: last run, last success, last failure, next update, projection counts, recovery action.
- Add i18n copy for en, pt-BR, and es.
- Update `docs/operators.md` with trust-list setup, sync behavior, failure handling, and recovery steps.
- Run final build/typecheck/validate-all.

### Out of Scope

- Separate `/admin/trust-lists/new` route.
- Full historical run drill-down.
- Executive dashboard/export; Phase 21 owns it.
- Legal/eIDAS trust-policy validation.

## Threat Model

- UI must not expose unmanaged groups to group-admin operations.
- UI must not imply preview is required when user selected optional test.
- UI must not imply legal trust-policy validation.
- Documentation must not contradict server-side authorization or recovery behavior.
