# Phase 20 Plan 20-01: Trust-List Operator Permissions, Preview, and Recovery Model

**Created:** 2026-04-23
**Status:** Ready for execution

## Objective

Add the server-side foundation for operator-friendly trust-list management: group-admin scoped authorization, non-mutating source preview/testing, recovery guidance mapping, and validator coverage.

## Scope

### In Scope

- Allow platform admins and group-admins to list/create/sync trust-list sources within their allowed group scope.
- Add a non-mutating source preview/test helper and API route.
- Add recovery guidance mapping for known trust-list onboarding/sync failures.
- Preserve HTTPS/localhost URL policy and XMLDSig integrity gate.
- Add validator coverage for permissions, preview non-mutation, recovery mapping, and route wiring.

### Out of Scope

- Full UI wizard implementation; Plan 20-02 owns the client/page changes.
- Full run-history drill-down; Phase 20 delivers a simple timeline.
- Legal/eIDAS trust-policy validation.
- Executive analytics.

## Threat Model

- Group-admins must never create or sync trust-list sources for unmanaged groups.
- Preview/test must not mutate sources, sync runs, snapshots, projections, certificates, or targets.
- Preview must use the same URL, size, parse, and XMLDSig safety gates as sync.
- Recovery guidance must not hide raw failure evidence; it should add recommended actions.
