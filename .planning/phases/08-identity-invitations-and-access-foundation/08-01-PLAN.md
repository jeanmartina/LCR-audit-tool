Build the authentication and authorization foundation for the v1.1 milestone so the product can support invitation-only access, multi-provider login, role-scoped group membership, and enforceable access boundaries.
Purpose: Downstream admin, reporting, and deployment phases depend on a stable identity model and permission contract; this phase locks those foundations before new UI and onboarding work builds on them.
Output: Auth domain schema and helpers, invitation lifecycle, provider wiring, permission enforcement primitives, and validation scripts for the identity foundation.
</objective>
