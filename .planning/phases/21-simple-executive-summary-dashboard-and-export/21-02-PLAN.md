# Phase 21 Plan 21-02: Executive Dashboard Route, Reporting Integration, and Print Surface

**Created:** 2026-04-23
**Status:** Ready for execution

## Objective

Deliver the executive-facing page, integrate it into the existing reporting flow, and add browser-print support.

## Scope

### In Scope

- Add a dedicated executive route, expected `/reporting/executive`.
- Link/toggle to it from `/reporting`.
- Render executive cards, top risks, upcoming issues, short trend, and breakdowns.
- Add browser-print support and evidence links.
- Add i18n strings for executive UI and print/export labels.

### Out of Scope

- Compose/runtime proof and docs closure; Plan 21-03 owns it.
- Full BI-style filtering and analytics.

## Threat Model

- The executive route must remain available to any authorized user, not admins only.
- The page must not degrade into the existing operational grid.
- Print output must not drop authorization-scoped context or key sections.
