# Phase 17 Plan: First-Run Bootstrap and Certificate/ZIP Onboarding UX

**Created:** 2026-04-20
**Status:** Ready for execution

## Objective

Remove first-use and certificate onboarding friction by replacing SQL/manual admin bootstrap, fake preview sections, and raw ZIP JSON responses with guided web flows that preserve the existing authorization and import pipeline.

## Scope

### In Scope

- first-run web setup for the initial platform admin
- setup API that creates the first platform admin, hashes password, creates a session, audits creation, and refuses after bootstrap
- single-certificate preview for fingerprint, derived CRLs, ignored/tracked split, and effective group defaults
- guided single-certificate import page using Phase 16 UI primitives
- guided ZIP import page using Phase 16 UI primitives
- import-run result page showing ZIP summary and per-file failures/results
- browser-safe relative redirects for touched POST routes
- i18n in `en`, `pt-BR`, and `es`
- dedicated Phase 17 validator wired into `scripts/validate-all.js`

### Out of Scope

- trust-list onboarding or ETSI parsing
- executive dashboards
- Entra/OIDC provider proof
- full real-time upload progress bars
- replacing all remaining inline styles outside the touched onboarding surfaces

## Threat Model

### Assets

- first platform-admin identity and password hash
- active auth session cookie
- uploaded certificate material
- group authorization boundaries
- import run evidence and audit trail

### Threats and Mitigations

- **Admin bootstrap remains open after setup**
  - Mitigation: `/setup` and `/api/setup/platform-admin` both check `hasPlatformAdmin`; POST re-checks immediately before create.
- **Weak or mismatched password creates unusable/admin-risk account**
  - Mitigation: require password confirmation and minimum 12 characters before calling `createFirstPlatformAdmin`.
- **Preview bypasses authorization**
  - Mitigation: `previewCertificateImport` runs the same group manage checks as `importCertificate`.
- **Preview result is trusted during commit**
  - Mitigation: commit route re-parses uploaded file and form values; preview is advisory only.
- **Raw JSON or internal-origin redirects leak implementation details**
  - Mitigation: browser form flows use relative `Location` headers and operator-facing result pages.
- **ZIP partial failures are hidden**
  - Mitigation: import-run result page shows every item result and message.
