## Milestone Summary

**Decimal Phases:**

- Phase 21.1: Public landing page and discoverable authentication entry (inserted after Phase 21 for an urgent published-root/login-discoverability fix)

**Key Decisions:**

- Decision: Extend trust-list ingestion through the existing certificate-first pipeline instead of inventing a second inventory model.
- Decision: Treat XMLDSig validation as a blocking gate before trust-list snapshots or imports are accepted.
- Decision: Make operator UX improvements incremental and grounded in shared server-safe primitives instead of a framework rewrite.
- Decision: Keep executive reporting authorized for normal users with group access, not admin-only.
- Decision: Preserve invite-gated provider semantics while making the published root URL a usable auth entry surface.

**Issues Resolved:**

- Replaced SQL/bootstrap friction with first-run web setup and guided onboarding.
- Added trust-list ingestion, sync auditing, change detection, provenance, preview, and recovery UX.
- Shipped a management-facing executive route with PDF/print output and packaged validation.
- Fixed the published-root 404/auth discoverability problem before milestone closure.

**Issues Deferred:**

- Full historical trust-list sync drill-down remains deferred.
- Legal/eIDAS trust-policy validation remains intentionally out of scope.
- Entra ID and generic OIDC live proof remain future milestone scope.
- Burn-rate/error-budget executive analytics remain future scope.

**Technical Debt Incurred:**

- Phases 16-17 still use legacy validation-strategy artifacts instead of normalized Nyquist frontmatter.
- Manual Docker/staging UAT is still recommended for executive print/PDF parity and the new root/auth entry path.
- Provider verification remains manually tracked by platform admins.
